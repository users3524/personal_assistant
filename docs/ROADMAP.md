# Roadmap

最后更新：2026-06-21

本文只写路线图和取舍，不把未来能力描述为当前已实现。

## 1. 新路线原则

| 原则 | 说明 |
| --- | --- |
| 闭环优先 | 已有表和服务必须先形成用户可见闭环，再继续加底座 |
| 本地优先 | 用户数据、图片、密钥和备份默认本地；云端 AI 只在明确场景调用 |
| 机会性后台 | Android WorkManager 只作为机会性触发，前台补偿必须可独立完成任务 |
| 证据链 | 高光、简历 bullet、RAG 引用都必须能追溯到本地事实 |
| 成本闸门 | 白天捕获保持短上下文，深夜生成有预算、重试上限和降级 |
| 输出隔离 | AI 只生成内容，不控制 Flutter/PDF 排版 |
| 安全可解释 | 备份、导入、AI 请求、向量重建都要有用户可理解的状态和失败原因 |

## 3. 近期路线

### R1：复盘生成闭环

目标：pending `review_generation_jobs` 可以被执行并写入日报。

| 交付 | 说明 |
| --- | --- |
| RawContextPackBuilder | 从待办、chat_turns、已有日报草稿、patting_logs 组包 |
| 前台执行入口 | App 打开后或复盘页手动触发 pending job |
| Runner 接入 | 调用 `NightlyStructuredReviewRunner`，复用 3 次调用上限 |
| 写入日报 | 成功写 `aiComment` / `aiSuggestion` 或创建待校准占位 |
| 状态 UI | 首页显示 pending、failed、calibrationRequired |
| WorkManager 回调 | 只调用同一入口；失败可被下次前台补偿 |

验收：无网络或模型失败时不会无限重试，不会丢 raw dump，不会静默写空日报。

### R2：高光确认闭环

目标：高光从“表和 DAO”变成用户可管理的证据池。

| 交付 | 说明 |
| --- | --- |
| 自动候选 | 每日最多 0-2 个候选，宁缺毋滥 |
| 高光收件箱 | 确认、忽略、编辑 |
| 来源追溯 | 跳转到日报、待办或文玩打卡 |
| 项目绑定 | 项目经历绑定多个高光并排序 |
| 备份恢复 | 高光和项目关系继续纳入镜像测试 |

验收：未确认高光不能进入正式简历，删除源数据不会留下孤儿关系。

### R3：简历输出闭环

目标：高光可以转化为可靠简历输出。

| 交付 | 说明 |
| --- | --- |
| PDF 导出 | `pdf` / `printing`，A4 模板，字体策略 |
| STAR 入口 | 基于项目事实和确认高光生成最多 3 条 bullet |
| 输出清洗 | 接入 `ResumeAiOutputSanitizer` |
| 极端文本测试 | 长 URL、无空格英文、中文长句、超长姓名 |
| 分享区分 | PNG 分享和 PDF 导出分开 |

验收：AI 不参与排版，PDF 关键文本可验证，极端文本不抛错。

## 4. 中期路线

### R4：RAG 与长期记忆

| 交付 | 说明 |
| --- | --- |
| EmbeddingJob | 对确认高光、日报摘要等高价值文本生成向量 |
| 重建入口 | 模型/维度变更时用户可重建 |
| 索引状态 | 缺失、不兼容、ready 可见 |
| 检索限窗 | Top-K <= 5，单片 <= 400 字 |
| 周报接入 | 只在周报/规划类场景使用，不进白天聊天 |

### R5：人生罗盘

| 交付 | 说明 |
| --- | --- |
| 五维目标持久化 | 固定维度，不做无限自定义 |
| 30 天冷却 | 已有策略服务，需 UI 和存储 |
| 存量任务迁移 | 候选建议 + 用户审核 |
| 任务绑定 | 根任务绑定维度，子任务继承展示 |
| 统计视图 | 维度投入与完成趋势 |

## 5. 暂缓或不做

| 项目 | 决策 |
| --- | --- |
| 文玩估值回归 | 不做。遗留表只兼容旧数据 |
| 新增 AI 底座表 | 暂缓，直到 R1-R3 完成 |
| 全量历史灌入 prompt | 不做，必须走限窗检索 |
| AI 生成排版/样式 | 不做，排版由 Flutter/PDF 模板控制 |
| 后台准点承诺 | 不做，只提供机会性后台 + 前台补偿 |
| 数据库物理删估值表 | 暂缓，除非单独迁移版本完整验证 |
