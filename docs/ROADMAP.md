# Roadmap

最后更新：2026-06-20

Roadmap 以“本地优先 + AI 低成本 + 长期资产化”为主线。文玩记录模块继续保留，不再作为默认下线项。

## 1. 当前里程碑

| 里程碑 | 状态 |
| --- | --- |
| v1.0.2 基线 | 已完成，包含待办、文玩、AI 复盘、简历、设置/备份。 |
| schemaVersion 6 | 已完成，包含清单、子任务、重复策略、软删除。 |
| 文档重构 | 进行中，本轮完成最新模块边界设计。 |

## 2. v1.1：白天数据捕获

| 优先级 | 工作 |
| --- | --- |
| P0 | 每日 AI 实时交互最多 15 轮。 |
| P0 | 熔断后进入离线文本便签模式。 |
| P0 | 单次文本输入最多 500 字。 |
| P0 | STT 单次录音最多 60 秒。 |
| P0 | 白天禁止历史日报/周报/RAG 检索。 |
| P1 | 高优任务完成后单轮教练式追问。 |
| P1 | 习惯打卡与最多 4 张图片路径。 |
| P1 | 文玩盘玩记录进入当日日报素材包。 |

## 4. v1.3：深夜日报引擎

| 优先级 | 工作 |
| --- | --- |

## 8. 暂不做

| 项 | 原因 |
| --- | --- |
| 白天长对话人格陪聊 | 与低 Token 捕获目标冲突。 |
| 白天 RAG 历史检索 | 成本和上下文不可控。 |
| 允许无限战略维度 | 会稀释人生罗盘权重。 |
| AI 控制 PDF 样式 | 会造成排版不可控。 |
| 删除文玩记录 | 用户明确要求保留。 |
