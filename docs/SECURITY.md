# 数据安全说明

## 一、当前方案（v1.0.1）

本应用定位为个人工具，数据存储于本地 SQLite 数据库，不上传任何服务器。

| 安全措施 | 状态 |
|----------|------|
| AI API Key 持久化加密 | 🟡 明文存入数据库（v1.1 计划加密） |
| 备份文件密码保护 | ❌ 已移除（v1.0.1），改为纯 JSON 明文 |
| 图片数据持久性 | ✅ Base64 内联 JSON，换设备/清缓存不丢失 |
| 数据传输安全 | ✅ 仅通过 HTTPS 调用 AI API |
| 网络请求 | ✅ 仅发往用户配置的 AI 供应商 |

## 二、备份与恢复

### 导出
- 所有数据序列化为 JSON
- `imagePaths`/`photoPaths` 中的图片 → 读取 → Base64 编码 → `base64:xxxxx` 格式
- 用户可选择保存到应用默认目录或通过 SAF 选择自定义位置

### 导入
- 读取 JSON → 逐表逐行使用 Raw SQL INSERT 恢复
- 检测 `base64:` 前缀 → 解码 → 写入临时目录
- 不支持的文件路径自动跳过

### 风险提示
- 备份文件为**明文 JSON**，请用户自行保管
- 导入前**覆盖**当前所有数据，不可撤销

## 三、敏感数据范围

| 数据类型 | 存储位置 | 敏感等级 |
|----------|----------|----------|
| AI API Key | `user_preferences.ai_api_key` | 🔴 高 |
| 用户个人信息 | `resume_profile` 表 | 🟡 中 |
| 藏品购入价格 | `antique_items` 表 | 🟡 中 |
| 备份文件 | 用户指定路径 | 🔴 高 |
| 复盘内容 | `daily_reviews` 表 | 🟢 低 |

## 四、开发规范

| 规范 | 说明 |
|------|------|
| API Key 不记录到日志 | commit/print 中不可出现 Key |
| 图片不保留 EXIF | 取图后直接读 bytes → Base64 |
| 构建产物不上传 | `.dart_tool/`、`build/`、`.reasonix/` 均在 `.gitignore` |
