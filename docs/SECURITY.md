# 数据安全与成本控制

最后更新：2026-06-20

本应用定位为本地优先的个人工具。用户数据默认存储在本机 SQLite 与应用沙盒中，只有用户配置并触发 AI 功能时，才向用户配置的 OpenAI 兼容 API 地址发送必要文本。

## 1. 当前安全现状

| 项 | 状态 |
| --- | --- |
| 本地数据库 | Drift + SQLite，本地保存。 |
| AI API Key | 当前在用户偏好表中持久化，后续需加密。 |
| AI 请求 | 仅发往用户配置的 API 地址。 |
| 备份 | 当前为明文 JSON，导出文件由用户自行保管。 |
| 图片 | 数据库保存路径列表，备份时可能编码为 `base64:`。 |
| 文玩数据 | 包含购入价格、估值、照片，按敏感数据处理。 |

## 2. 敏感数据范围

| 数据 | 敏感等级 | 存储位置 |
| --- | --- | --- |
| AI API Key | 高 | `user_preferences.ai_api_key` |
| 简历个人信息 | 高 | `resume_profile` |
| 每日复盘与对话素材 | 高 | `daily_reviews`、未来 `chat_turns` |
| 文玩照片、购入价、估值 | 中高 | `antique_items`、`valuation_records`、沙盒图片 |
| 待办任务 | 中 | `todos` |
| 备份文件 | 高 | 用户选择位置 |

## 3. 白天 AI 成本控制

| 控制点 | 规则 |
| --- | --- |
| 每日实时对话 | 最多 15 轮。 |
| 熔断后 | 停止云端请求，只保存离线便签。 |
| 文本输入 | 单次最多 500 字。 |
| 语音输入 | 单次最多 60 秒。 |
| Prompt 上下文 | 控制在 2K tokens 内。 |
| 历史检索 | 白天禁止 RAG 和历史日报检索。 |

## 4. 深夜 AI 成本控制

| 控制点 | 规则 |
| --- | --- |
| 触发时间 | 2:00-5:00。 |
| 设备条件 | 充电 + Wi-Fi。 |
| 原始文本 | 打包最多 8000 字。 |
| Input tokens | 控制在 10K 内。 |
| 结构化解析失败 | 最多重试 2 次。 |
| 连续失败 | 停止重试，保存原文并标记待手动校准。 |

## 5. 备份与恢复风险

当前备份服务会导出多张表，并尝试将图片路径对应文件编码为 `base64:` 字符串。

| 风险 | 说明 |
| --- | --- |
| 明文泄露 | JSON 备份可能包含 API Key、简历、复盘、文玩价格。 |
| 文件过大 | 图片 Base64 可能导致备份文件变大。 |
| 路径失效 | 其他设备的绝对路径可能不可用。 |
| 覆盖恢复 | 导入前会清空当前数据，需谨慎操作。 |

长期目标是升级为专有备份包：数据库文件 + 图片目录 + manifest，必要时再加入密码保护。

## 6. 开发规范

1. 不在日志、截图、提交信息中暴露 API Key。
2. 不默认上传图片内容给 AI。
3. 不将白天便签和历史日报混入实时 Prompt。
4. 不用无限重试消耗 Token。
5. 不在未经确认时删除文玩记录、估值记录或复盘历史。
6. 任何数据迁移必须可解释、可测试、可回滚或至少可备份。
