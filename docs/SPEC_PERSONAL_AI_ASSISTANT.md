## 1. 模块一：数据捕获与日常对话

本模块作为系统全天候的数据捕获网口，核心职责是在本地以最低算力与 Token 消耗捕获用户全天行为碎片。

### 1.1 核心功能

| 功能项 | 业务逻辑规范 | 输入与存储限制 |
| --- | --- | --- |
| 待办项创建 | 支持创建常规待办。默认 `startedAt` 为当前时间，`dueDate` 可选。 | 标题最多 100 字，描述最多 1000 字。 |
| 习惯打卡记录 | 记录特定习惯的即时完成状态，支持单次多张图片相对路径挂载。 | 单个习惯单次打卡图片上限 4 张。 |
| 文玩记录 | 保留藏品档案、盘玩打卡、照片、估值、榜单。 | 文玩数据默认本地保存，深夜作为日报素材。 |
| 教练式对话引导 | 捕获器不开启发散闲聊。高优任务完成时触发单轮追问，捕获核心突破或卡点。 | Prompt 上下文严格控制在 2K tokens 以内。 |

### 1.2 硬性边界

| 边界 | 规则 |
| --- | --- |
| 对话轮数强制熔断 | 单日实时 AI 交互最高 15 轮。达到后切断云端 API。 |
| 熔断 UI | 对话框转为只收不发的离线文本便签模式。 |
| 熔断提示 | “今日复盘素材已足够，深夜将为您合并生成深度日报”。 |
| 单次文本输入 | 最多 500 字。 |
| STT 语音输入 | 单次录音最长 60 秒，超时自动截断并识别。 |
| 全天候零长上下文 | 白天 AI 请求只携带当天对话上下文和当天待办列表。 |
| 禁止行为 | 白天禁止读取历史日报、周报或进行向量数据库 RAG 检索。 |

## 2. 模块二：异步深夜精炼与日报生成

本模块在深夜后台无人值守时运行，通过单次高算力调用，将白天结构化数据与零散文本融合成深度历史档案。

### 2.1 核心功能

| 功能项 | 业务逻辑规范 | 数据流向 |
| --- | --- | --- |
| 定时任务打包 | 捞取本地 SQLite 中当日 Todo、文玩打卡、习惯打卡、15 轮内原始对话文本。 | 组装为固定结构化 JSON。 |
| 多维深度复盘 | 调用高参数云端大模型，从执行进度、情绪状态、认知卡点、意外收获四维生成第二人称日报。 | Markdown 写入 `daily_reviews`。 |
| 高光特征抽离 | 使用结构化输出，强制模型返回是否触发高光里程碑及摘要。 | 对应待办、打卡或里程碑打上 `#简历素材`。 |

### 2.2 硬性边界

| 边界 | 规则 |
| --- | --- |
| 后台触发窗口 | 严格限定凌晨 2:00 至 5:00。 |
| 设备条件 | 手机必须正在充电且已连接 Wi-Fi。 |
| 顺延策略 | 条件不满足时，顺延至清晨用户首次打开 App 后低优先级静默执行。 |
| 原始素材裁剪 | 白天原始文本最多打包 8000 字。 |
| 裁剪优先级 | 优先保留 Todo 成果和教练对话，剔除冗余随笔。 |
| Input Token | 深夜大模型输入稳控在 10K tokens 内。 |
| 解析异常 | 结构化输出 Parser 报错最多重试 2 次。 |
| 降级策略 | 连续失败后终止重试，保存原始文本，结构化得分标记为“待手动校准”。 |

## 3. 模块三：长效记忆与人生罗盘规划

本模块通过向量化持久层将离散日报连接成网，使 AI 具备大跨度时间视角，完成长期目标对齐。

### 3.1 核心功能

| 功能项 | 业务逻辑规范 | 数据与算力指标 |
| --- | --- | --- |
| 文本向量化持久层 | 模块二生成的深度日报转化为数值向量。 | 使用标准 1536 维向量，写入本地轻量向量存储。 |
| 长期人生罗盘 | 顶层战略框架包含职业技术、身心健康、个人爱好、财务财富、社会关系。 | 所有底层待办必须强绑定五维之一。 |
| 跨天滚存状态机 | 未完成任务跨天时物理创建时间不变，展示层和统计层自动顺延。 | 顺延逻辑完全基于只读 `displayDate`。 |

### 3.2 硬性边界

| 边界 | 规则 |
| --- | --- |
| 战略维度数量 | 人生罗盘宏观责任领域最多且必须为 5 个。拒绝第 6 个维度。 |
| 目标修改冷却 | 长期目标和愿景保存后，每 30 天内仅允许修改一次。 |
| RAG Top-K | 周报或人生规划纠偏时最多检索 5 条历史记忆切片。 |
| 单条切片 | 每条最多 400 字。 |
| Prompt 总量 | 周报整体 Prompt 控制在 12K tokens 内。 |
| 禁止行为 | 不允许把历史日报全量读入 Prompt。 |

## 4. 模块四：高光摘取与 STAR 简历生成

本模块负责将日常积累的高价值高光数据转化为符合招聘场景的简历资产。

### 4.1 核心功能

| 功能项 | 业务逻辑规范 | 技术实现 |
| --- | --- | --- |
| 技术栈徽章化 | `project_experiences.tech_stack` 使用 `StringListConverter` 保存字符串数组。 | UI 支持逗号分隔输入，渲染层用 `Wrap + Chip`。 |
| 高光记录交叉筛选 | 支持关键词及时间过滤，一键从本地关系型数据库中捞出命中标签的成就。 | 基于外键与 Tag 索引查询，零网络消耗。 |
| STAR 叙事润色 | 将零碎、口语化里程碑文本重构为招聘场景业绩描述。 | 大模型输出紧凑圆点列表纯文本。 |

### 4.2 硬性边界

| 边界 | 规则 |
| --- | --- |
| 里程碑判定门槛 | 拒绝日常无价值行为进入高光库。 |
| 单篇日报高光上限 | 自动捕获高光最多 2 条；无重大突破则为 0。 |
| 项目经历描述行数 | STAR Bullet Points 最多 3 条。超出部分持久化时丢弃。 |
| 样式边界 | AI 仅输出纯文本，绝不参与 PDF / 页面排版样式。 |
| 排版责任 | A4 规整排版、字号、间距全部由 Flutter 模板硬编码控制。 |

## 5. 数据基础设施与核心代码架构规范

### 5.1 待办四层级自关联约束

所有任务必须遵循多层级树状设计，子任务通过 `parentId` 自关联父任务。

| 铁律 | 规则 |
| --- | --- |
| 级联状态联动 | 父任务被标记为 done 或 cancelled 时，DAO 必须自动将其下所有子任务变更为相同状态。 |
| 软删除级联 | 父任务 `deletedAt != null` 后，其下所有子任务在所有查询结果中对用户隐形。 |

### 5.2 状态层树状组装

Repository 层向 Riverpod 状态层吐数据时，禁止直接返回数据库打平数组。

```dart
// Step 1: 查询 parentId == null 且符合今日活跃展示的父任务。
// Step 2: 提取父任务 ID 集合，执行 IN 查询一次性批量捞出子任务行。
// Step 3: 在 Dart 内存中 groupBy parentId，将子任务注入父任务 subtasks。
```

### 5.3 仪表盘统计分母校准

今日任务总数必须剔除子任务，采用逻辑并集：

```text
今日总数 = 今天新创建的父任务 ∪ 历史创建且至今未完成、未取消的父任务
```

底层 Drift SQL 必须强制包含：

```sql
WHERE parent_id IS NULL
```

子任务完成度只表现为父任务卡片内部局部进度，例如 `2/4`，绝不计入主仪表盘分母。

## 6. 文玩记录保留声明

文玩记录是本项目的重要生活记录模块，保留范围包括：

1. `antique_items` 藏品主表。
2. `patting_logs` 盘玩打卡日志。
3. `valuation_records` 估值历史。
4. 分类、子类、专属字段与图片路径。
5. 趣味榜单与盘玩时间线。

后续任何体积优化、图表替换或 UI 改造，都不得未经确认删除用户文玩数据。
